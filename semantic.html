Semantic HTML

Why semantic markup matters for accessibility and SEO

❌ Non-Semantic (Bad)

<div class="article">
  <div class="header">
    <div class="title">Article Title</div>
  </div>
  <div class="content">
    <p>Article text...</p>
  </div>
  <div class="footer">
    <span>Author</span>
  </div>
</div>

✓ Semantic (Good)

<article>
  <header>
    <h1>Article Title</h1>
  </header>
  <section>
    <p>Article text...</p>
  </section>
  <footer>
    <address>Author</address>
  </footer>
</article>

❌ Non-Semantic (Bad)

<div class="menu">
  <div class="link">
    <a href="/">Home</a>
  </div>
  <div class="link">
    <a href="/about">About</a>
  </div>
</div>

✓ Semantic (Good)

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
  </ul>
</nav>

❌ Non-Semantic (Bad)

<div role="button" 
     onclick="submit()">
  Click Me
</div>

✓ Semantic (Good)

<button type="button"
        onclick="submit()">
  Click Me
</button>

❌ Redundant ARIA (Bad)

<button role="button">
  Click Me
</button>

✓ Clean Semantic (Good)

<button>
  Click Me
</button>

❌ Wrong Element (Bad)

<a role="button" 
   onclick="submit()">
  Submit
</a>

✓ Right Elements (Good)

<!-- For actions -->
<button>Submit</button>

<!-- For navigation -->
<a href="/page">Go to Page</a>

Why Use Semantic HTML?

  • Accessibility: Screen readers can understand page structure and navigate easily
  • SEO: Search engines better understand your content's meaning and context
  • Maintainability: Code is easier to read and understand for developers
  • Keyboard Navigation: Semantic elements like buttons have built-in keyboard support
  • Standards: Following web standards ensures better browser compatibility
